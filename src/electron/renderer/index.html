New app

Fill this in and kube-helper does the Docker work for you — downloading, starting, and keeping it running. Builds a kh/v1 App manifest and runs it through the same reconciler as kh apply.

A short name you'll recognise. Lowercase letters, numbers and dashes only. DNS-1123 label (lowercase alphanumerics and dashes). Used for container names kh-<name>-<i> and the network alias.

Think of it as the app's installer — it's downloaded automatically from Docker Hub. Pick a suggestion, or paste any image name a developer gave you. Any registry reference (repo:tag or host/repo@sha256:…); pulled if absent locally.

How many identical copies run at once. With 2 or more, the app stays up even if one copy crashes, and updates happen with no downtime. Desired replica count (0–100). Each is a container kh-<name>-<i>; scaling is reconciled per index without disturbing existing replicas.

Overrides the image's default command. Parsed as a shell-style argv (quotes respected). Leave blank to keep the image default.

You'll open the app at localhost:<port>. Each extra copy uses the next port up. Bound on the host; replica i gets host+i so replicas never collide.

Where the app listens internally — web servers are usually 80. The suggestion list's apps use their defaults. The port the process listens on inside the container.

Passed to the app as environment variables — e.g. a database needs POSTGRES_PASSWORD = something-secret. Environment variables passed to every replica.

Absolute, or relative to where you'd run kh apply.

Files saved there survive updates, restarts and crashes. For PostgreSQL use /var/lib/postgresql/data, for MySQL /var/lib/mysql. Absolute container path. Managed volumes are per-replica so stateful replicas never share a data dir.

kube-helper pings the app every few seconds; copies that stop answering are marked unhealthy and restarted by the watcher. Compiles to a Docker HEALTHCHECK; readiness (status READY) and kh watch restarts key off it.

Docker restart policy for crashing processes — orthogonal to kh watch, which handles removed/stopped/unhealthy replicas.

Logs

What the app prints while running — the first place to look when something misbehaves. Each line is tagged with the copy it came from; errors are red.
